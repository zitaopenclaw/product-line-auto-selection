=== TASK ===
For EACH candidate, score how well the customer need maps to that hardware catalog node.

Guidelines:
- Business Group is a HARD context here, not a soft signal. All candidates are already filtered to {business_group} hardware — do not penalise for being in the wrong BG.
- Prefer more specific (deeper) nodes when the description gives enough detail. A confident L4 model match beats a vague L2 brand match.
- A node is a good match if the hardware product line catalogued under it would satisfy the described customer need — even if only a subset of variants applies.

- 1.00 = The node is the exact, clearly-implied answer (the description names this specific product line or model).
- 0.85–0.99 = Strong match. The node's product domain directly and specifically addresses the described need.
- 0.60–0.84 = Plausible match. The node covers the right area but the description is not specific enough to be certain.
- 0.30–0.59 = Weak / tangential. Some overlap in terminology but this node is not a real fit.
- 0.00–0.29 = No real match. Different product category entirely. Skip if possible.
